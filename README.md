# OpenCart E-Commerce QA Project

## Overview

Comprehensive QA project conducted on the OpenCart E-Commerce platform covering Manual Testing, Automation Testing, API Testing, and Database Testing.

As Team Leader, I participated in planning, execution, defect management, and automation framework implementation.

---

## Technologies Used

- Java
- Selenium WebDriver
- Cucumber BDD
- TestNG
- Maven
- Page Object Model (POM)
- Postman
- SQL Server
- Jira
- Git & GitHub

---

## Testing Activities

### Manual Testing

- Test Planning
- Test Scenario Design
- Test Case Design
- Functional Testing
- Regression Testing
- Smoke Testing
- Bug Reporting

### Automation Testing

Implemented a Selenium-Cucumber framework using:

- Page Object Model (POM)
- Feature Files
- Step Definitions
- Hooks
- Test Runners
- Assertions
- Screenshots on Failure

### Automated Scenarios

- User Registration
- User Login
- Product Search
- Add to Cart
- Checkout
- Logout

### API Testing

Performed API validation using Postman:

- GET Requests
- POST Requests
- PUT Requests
- DELETE Requests
- Status Code Validation
- Response Validation
- Authentication Testing

### Database Testing

Validated backend data using SQL:

- User Creation Validation
- Order Validation
- Product Inventory Validation
- Data Integrity Checks
